Project 02

Greeks Curves Dashboard

Interactive Python dashboard that visualizes option prices and Greeks as functions of the underlying price, combining Black-Scholes, Monte Carlo pricing, Yahoo Finance option chains, and a smile-adjusted volatility proxy.

Live demo View full code

1. Project Objective

The purpose of this project is to build a derivatives risk dashboard focused on visualizing how option prices and Greeks evolve when the underlying price changes. Instead of returning a single static Greek value, the application generates full curves across a configurable spot grid and adds reference markers for spot, strike, forward price, and barrier levels.

  • Input: manual option parameters or Yahoo Finance option chains.
  • Pricing: analytical Black-Scholes for vanilla products and Monte Carlo simulation for exotic payoffs.
  • Risk analytics: first-, second-, and third-order Greeks, including Delta, Gamma, Vega, Vanna, Volga, Charm, Speed, Color, Zomma, Veta, Vera, and Ultima.
  • Output: current price/Greeks, interactive Greek curves, scenario comparison, and volatility-smile diagnostics.
Stack Python, Streamlit, NumPy, Pandas, Plotly, SciPy, yfinance
Products Vanilla, digital, barrier, gap, and capped options
Use case Derivative risk visualization and scenario analysis

2. Greeks Coverage and Risk Taxonomy

The dashboard organizes risk sensitivities by order. This lets the user choose whether to focus on standard first-order risk, convexity and cross-Greeks, or more advanced third-order sensitivities. Price is always included as the baseline output.

def build_selected_greek_names(include_first_order=True, include_second_order=True, include_third_order=True):
    selected = {"Price"}

    if include_first_order:
        selected.update(FIRST_ORDER_GREEKS)
    if include_second_order:
        selected.update(SECOND_ORDER_GREEKS)
    if include_third_order:
        selected.update(THIRD_ORDER_GREEKS)

    return [greek_name for greek_name in GREEK_NAMES if greek_name in selected]

3. Yahoo Finance Data Pipeline

For market-based inputs, the application loads the underlying spot and listed option chains from Yahoo Finance. Quotes are cleaned by removing expired maturities, invalid strikes, unusable bid-ask pairs, and empty mid-prices before they are used for calibration or pricing.

app.py - option chain cleaning 
def load_option_data(symbol, quote_option_type):
    ticker = yf.Ticker(symbol)
    rows = []
    today = datetime.today()

    try:
        maturities = ticker.options
    except Exception:
        maturities = []

    for maturity_string in maturities:
        maturity_dt = datetime.strptime(maturity_string, "%Y-%m-%d")

        if maturity_dt <= today + timedelta(days=1):
            continue

        chain = ticker.option_chain(maturity_string)
        chain_df = pd.DataFrame(chain.calls if quote_option_type == "Calls" else chain.puts)
        chain_df["option_type"] = quote_option_type
        chain_df["maturity"] = maturity_string
        rows.append(chain_df)

    data = pd.concat(rows, ignore_index=True)
    data["mid"] = (pd.to_numeric(data["bid"], errors="coerce") + pd.to_numeric(data["ask"], errors="coerce")) / 2
    data = data.dropna(subset=["strike", "bid", "ask", "mid"])
    data = data[(data["strike"] > 0) & (data["bid"] >= 0) & (data["ask"] > 0) & (data["mid"] > 0)]

    return data.reset_index(drop=True)

The pricing setup includes risk-free rates and dividend yield rather than assuming a zero-carry framework. The application pulls currency-aware rates when possible and estimates dividends from trailing Yahoo Finance dividend data or from the platform dividend-yield field.

5. Pricing Engines: Black-Scholes and Monte Carlo

Vanilla calls and puts are priced with Black-Scholes. Non-vanilla payoffs are handled with a Monte Carlo engine that simulates risk-neutral geometric Brownian motion, evaluates terminal or path-dependent payoffs, and discounts the expected payoff back to present value.

app.py - pricing engines 
def vanilla_price(S, K, tau, sigma, r, q, option_type):
    d1 = (np.log(S / K) + (r - q + 0.5 * sigma**2) * tau) / (sigma * np.sqrt(tau))
    d2 = d1 - sigma * np.sqrt(tau)

    if option_type == "Calls":
        return S * np.exp(-q * tau) * norm.cdf(d1) - K * np.exp(-r * tau) * norm.cdf(d2)

    if option_type == "Puts":
        return K * np.exp(-r * tau) * norm.cdf(-d2) - S * np.exp(-q * tau) * norm.cdf(-d1)

def monte_carlo_price_single_batch(parameters, num_simulations, seed):
    n_steps = max(1, int(np.ceil(tau * FIXED_STEPS_PER_YEAR)))
    dt = tau / n_steps
    rng = np.random.default_rng(seed)
    z = rng.standard_normal((int(num_simulations), n_steps))

    log_returns = (risk_free_rate - dividend_yield - 0.5 * volatility**2) * dt + volatility * np.sqrt(dt) * z
    paths = spot * np.exp(np.cumsum(log_returns, axis=1))
    terminal_spot = paths[:, -1]

    return np.exp(-risk_free_rate * tau) * np.mean(np.asarray(payoffs, dtype=float))

7. Finite-Difference Greeks Engine

The Greeks are estimated by bumping spot, volatility, interest rates, and time to maturity. The engine caches Monte Carlo prices for repeated parameter combinations, which avoids recomputing the same scenario multiple times when higher-order Greeks require nested finite differences.

8. Smile-Adjusted Volatility Proxy

The dashboard can derive vanilla implied volatility points from option mid-prices using a Brent root solver. For exotic products, it builds a pragmatic smile-adjusted volatility proxy by combining the relevant vanilla volatility points: strike, spot, barrier, payoff strike, cap, or floor.

app.py - implied volatility and smile proxy 
def implied_volatility_from_mid(row, spot, risk_free_rate, dividend_yield):
    strike = safe_float(row.get("strike"))
    mid = safe_float(row.get("mid"))
    maturity = row.get("maturity")
    option_type = row.get("option_type")

    def objective(volatility):
        return vanilla_price(spot, strike, tau, volatility, risk_free_rate, dividend_yield, option_type) - mid

    return float(brentq(objective, 1e-6, 5.0, maxiter=200)), "Brent root from bid/ask mid"

9. Greek Curve Construction

The main analytical output is a curve dataframe built by repricing the same option across a grid of underlying prices. Each point stores the option price and the selected Greeks, which are then displayed in Plotly tabs with reference lines for the current spot, strike, forward level, and barrier.

app.py - curve dataframe and Plotly chart 
def build_greek_curves(parameters, s_min, s_max, number_of_points, num_simulations, greek_names=None):
    spot_values = np.linspace(float(s_min), float(s_max), int(number_of_points))
    rows = []

    for spot_value in spot_values:
        scenario_parameters = parameters_for_spot(parameters, spot_value)
        result = calculate_result(scenario_parameters, num_simulations, greek_names=greek_names)
        row = {"Spot": spot_value}
        row.update(tuple_to_dict(result))
        rows.append(row)

    return pd.DataFrame(rows)

def create_greek_curve_figure(curve_df, greek_name, current_spot, strike, forward_price, barrier=None, comparison_curve_df=None):

10. Scenario Analysis Interface

After generating the base curves, users can open an interactive scenario tab and modify spot, strike, maturity, volatility, rates, dividend yield, barrier, or extra strike. The app recalculates a new curve and overlays it against the base case, making the dashboard useful for stress testing and sensitivity analysis.
