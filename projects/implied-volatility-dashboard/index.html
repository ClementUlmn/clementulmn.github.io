Project 02

Implied Volatility Dashboard

Interactive Python dashboard to determine and visualize implied volatility surfaces across option strikes and maturities using Black-Scholes, Newton-Raphson, and Yahoo Finance data.

Live demo View full code

1. Project Objective

The goal of this project is to build an interactive options analytics dashboard that retrieves market option-chain data, computes implied volatility, and displays the volatility surface across strikes and maturities.

  • Input: listed option chains from Yahoo Finance.
  • Core method: Black-Scholes pricing combined with Newton-Raphson calibration.
  • Output: volatility skew, term structure, and 3D implied volatility surface.

2. Option Chain Retrieval

The dashboard starts by downloading all available option maturities for a selected ticker. It extracts the main market quotes and computes the mid-price, which is used as the market reference price for implied volatility calibration.

app.py - option chain extraction 
def find_data(ticker, option):
    dfs = []
    maturities = ticker.options

    for maturity in maturities:
        chain = ticker.option_chain(maturity)

        if option == "Calls":
            df = pd.DataFrame(chain.calls)
            df["option_type"] = "Calls"

        elif option == "Puts":
            df = pd.DataFrame(chain.puts)
            df["option_type"] = "Puts"

        df = df[["strike", "bid", "ask", "currency", "option_type"]]
        df["maturity"] = maturity
        dfs.append(df)

    data = pd.concat(dfs, ignore_index=True)
    data["mid"] = (data["bid"] + data["ask"]) / 2
    return data

3. Risk-Free Rate Selection

The model selects a risk-free rate depending on the option currency. For example, USD options use SOFR, while EUR options use the euro short-term rate.

app.py - risk-free rate 
def risk_free_rate(data):
    currency = data["currency"].iloc[0]

    if currency == "USD":
        url = "https://fred.stlouisfed.org/graph/fredgraph.csv?id=SOFR"
        df = pd.read_csv(url)
        rf = float(df["SOFR"].iloc[-1]) / 100

    if currency == "EUR":
        url = "https://fred.stlouisfed.org/graph/fredgraph.csv?id=ECBESTRVOLWGTTRMDMNRT"
        df = pd.read_csv(url)
        rf = float(df["ECBESTRVOLWGTTRMDMNRT"].iloc[-1]) / 100

    return rf

4. Black-Scholes Pricing Engine

The Black-Scholes function prices calls and puts and computes option sensitivities such as delta, gamma, vega, theta, rho, volga, and vanna.

app.py - Black-Scholes pricer 
def pricer_bs(strike, maturity, IV, spot, rf, option_type):
    T = (maturity - datetime.today()).days / 365.0

    d1 = (np.log(spot / strike) + (rf + 0.5 * IV**2) * T) / (IV * np.sqrt(T))
    d2 = d1 - IV * np.sqrt(T)

    if option_type == "Calls":
        price = spot * norm.cdf(d1) - strike * np.exp(-rf * T) * norm.cdf(d2)
        delta = norm.cdf(d1)

    if option_type == "Puts":
        price = strike * np.exp(-rf * T) * norm.cdf(-d2) - spot * norm.cdf(-d1)
        delta = norm.cdf(d1) - 1

    gamma = norm.pdf(d1) / (spot * IV * np.sqrt(T))
    vega = spot * norm.pdf(d1) * np.sqrt(T)

    return price, delta, gamma, vega

5. Implied Volatility Calibration

Implied volatility is obtained by solving for the volatility that makes the Black-Scholes price match the observed market mid-price. The project uses Newton-Raphson because vega provides the derivative needed for fast convergence.

app.py - Newton-Raphson solver 
def newton_raphson(ticker, data, initial_IV, spot, rf, max_iter=100):
    IV = initial_IV
    mid = data["mid"]
    strike = data["strike"]
    maturity = datetime.strptime(data["maturity"], "%Y-%m-%d")

    for _ in range(max_iter):
        price, delta, gamma, vega, theta, rho, volga, vanna = pricer_bs(
            strike, maturity, IV, spot, rf, data["option_type"]
        )

        if abs(price - mid) <= 1e-5:
            return IV

        if abs(vega) < 1e-8:
            return np.nan

        IV = IV - (price - mid) / vega

    return np.nan

6. Volatility Skew and Term Structure

The dashboard can isolate one maturity to display the volatility skew across strikes, or isolate one strike to display the term structure across maturities.

app.py - skew and term structure 
# Skew: implied volatility across strikes for one maturity
data_maturity = data[data["maturity"] == maturity].copy()
data_maturity["model_IV"] = data_maturity.apply(
    lambda row: newton_raphson(ticker, row, 0.2, spot, rf),
    axis=1
)

# Term structure: implied volatility across maturities for one strike
data_strike = data[data["strike"] == strike].copy()
data_strike["model_IV"] = data_strike.apply(
    lambda row: newton_raphson(ticker, row, 0.2, spot, rf),
    axis=1
)

8. Interactive Streamlit Interface

The Streamlit interface lets users select the ticker, option type, strikes, maturities, and analysis mode without editing the Python code.

app.py 
def streamlit_app():
