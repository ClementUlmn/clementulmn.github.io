Project 01

Monte Carlo Pricer

Python derivatives pricer that benchmarks Black-Scholes and Monte Carlo across vanilla and exotic options, highlighting model discrepancies, Greeks exposure, payoff/risk profiles, and the impact of a volatility-skew model.

Live demo View full code

1. Project Objective

The goal of this project is to build a complete option-pricing workstation where users can select market data or manual inputs, choose a pricing methodology, and obtain both theoretical prices and risk sensitivities in a single Streamlit interface.

  • Input: Yahoo Finance option chains or fully manual parameters.
  • Core models: Black-Scholes closed-form pricing and Monte Carlo simulation.
  • Output: option price, Greeks, payoff diagram, Monte Carlo paths, and model comparison.
  • Coverage: vanilla calls and puts, digital options, barrier options, gap options, and capped options.

2. Market Data and Quote Cleaning

The application retrieves listed option chains from Yahoo Finance, filters out unusable maturities and illiquid quotes, and computes the bid-ask mid-price used as the market reference for volatility calibration.

app.py - option chain extraction 
def find_data(ticker, option):
    dfs = []
    maturities = ticker.options
    today = datetime.today()

    for maturity in maturities:
        maturity_datetime = datetime.strptime(maturity, "%Y-%m-%d")

        if maturity_datetime <= today + timedelta(days=1):
            continue

        chain = ticker.option_chain(maturity)

        if option == "Calls":
            df = pd.DataFrame(chain.calls)
            df["option_type"] = "Calls"
        elif option == "Puts":
            df = pd.DataFrame(chain.puts)
            df["option_type"] = "Puts"

        df = df[["strike", "bid", "ask", "currency", "option_type"]]
        df["maturity"] = maturity
        dfs.append(df)

    data = pd.concat(dfs, ignore_index=True)
    data["mid"] = (data["bid"] + data["ask"]) / 2
    data = data.dropna(subset=["strike", "bid", "ask", "mid"])
    return data[data["mid"] > 0]

3. Risk-Free Rate and Dividend Inputs

The pricing engine uses a currency-aware risk-free rate and automatically estimates the dividend yield from Yahoo Finance. This allows the model to use a cost-of-carry framework rather than assuming a zero-dividend underlying.

app.py - rates and dividends 
def risk_free_rate(data):
    currency = data["currency"].iloc[0]

    if currency == "USD":
        url = "https://fred.stlouisfed.org/graph/fredgraph.csv?id=SOFR"
        df = pd.read_csv(url)
        rf = float(df[df["SOFR"] != "."]["SOFR"].iloc[-1]) / 100

    if currency == "EUR":
        url = "https://fred.stlouisfed.org/graph/fredgraph.csv?id=ECBESTRVOLWGTTRMDMNRT"
        df = pd.read_csv(url)
        rf = float(df[df["ECBESTRVOLWGTTRMDMNRT"] != "."]["ECBESTRVOLWGTTRMDMNRT"].iloc[-1]) / 100

    return rf

4. Black-Scholes Pricing Engine

The Black-Scholes module prices standard calls and puts analytically. It also computes the main first-order and second-order sensitivities used in risk management: delta, gamma, vega, theta, rho, volga, and vanna.

app.py - Black-Scholes core 
def vanilla_price(S, K, tau, sigma, r, q, opt):
    d1 = (np.log(S / K) + (r - q + 0.5 * sigma**2) * tau) / (sigma * np.sqrt(tau))
    d2 = d1 - sigma * np.sqrt(tau)

    if opt == "Calls":
        return S * np.exp(-q * tau) * norm.cdf(d1) - K * np.exp(-r * tau) * norm.cdf(d2)

    if opt == "Puts":
        return K * np.exp(-r * tau) * norm.cdf(-d2) - S * np.exp(-q * tau) * norm.cdf(-d1)

gamma = np.exp(-q * T) * norm.pdf(d1) / (spot * IV * np.sqrt(T))
vega = spot * np.exp(-q * T) * norm.pdf(d1) * np.sqrt(T)

5. Monte Carlo Simulation Engine

The Monte Carlo engine simulates geometric Brownian motion paths under the risk-neutral measure. It discounts the average payoff and supports a large number of simulations with batching for performance and stability.

app.py - Monte Carlo paths 
def simulate_paths(S0, tau, sigma, r, q, rng_seed):
    n_steps = max(1, int(np.ceil(tau * steps_per_year)))
    dt = tau / n_steps

    rng = np.random.default_rng(rng_seed)
    Z = rng.standard_normal((num_simulations, n_steps))

    log_returns = (r - q - 0.5 * sigma**2) * dt + sigma * np.sqrt(dt) * Z
    S_paths = S0 * np.exp(np.cumsum(log_returns, axis=1))
    S_paths = np.column_stack((np.full(num_simulations, S0), S_paths))

    return S_paths

price = np.exp(-r * tau) * np.mean(payoffs)

7. Smile-Adjusted Volatility Proxy

For Yahoo Finance inputs, the application does not rely directly on the platform's implied volatility field. It solves vanilla implied volatilities from bid-ask mid-prices and can build a smile-adjusted proxy for exotic products by combining the relevant volatility points.

app.py - volatility proxy 
def get_quote_implied_volatility(ticker, quote_row, spot, rf, dividend_yield):
    # Every vanilla IV used by the app is solved from the bid/ask mid via Newton-Raphson.
    implied_volatility = newton_raphson(
        ticker, quote_row, 0.2, spot, rf,
        dividend_yield=dividend_yield
    )

    if np.isfinite(implied_volatility) and implied_volatility > 0:
        return float(implied_volatility), "Newton-Raphson from vanilla mid"

    return np.nan, "Unavailable from Newton-Raphson"

9. Interactive Streamlit Interface

The Streamlit interface allows the user to select the data source, pricing method, option type, maturity, strike, volatility mode, and number of Monte Carlo simulations without modifying the Python code.
